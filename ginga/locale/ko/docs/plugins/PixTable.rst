
``PixTable`` 은 어떤 영역의 픽셀 값을 확인하거나 모니터링하는 방법을 제공합니다.

**플러그인 유형: 로컬**

``PixTable`` 은 로컬 플러그인으로, 채널에 연결되어 있습니다.
각 채널마다 하나의 인스턴스를 열 수 있습니다.

**기본 사용**

가장 기본적인 사용에서는 단순히 채널 뷰어에서 커서를 움직이면 됩니다. 그러면
플러그인 UI의 「Pixel Values」 표시에 픽셀 값 배열이 나타납니다. 중앙 값이
강조되며, 이것은 커서 아래의 값에 해당합니다.

가장 왼쪽 콤보 상자 컨트롤에서 3x3, 5x5, 7x7 또는 9x9 격자를 선택할 수 있습니다.
배열 값이 측면에서 잘리지 않도록 「글꼴 크기」 컨트롤을 조정하면 도움이 될 수
있습니다. 표를 더 많이 보려면 플러그인 작업 공간을 크게 할 수도 있습니다.

.. note:: 이미지가 뒤집히거나 전치되거나 회전된 경우, 표시되는 값 표의 순서가
          반드시 채널 뷰어와 일치하지는 않습니다.

**마크 사용**

마크를 설정하고 선택하면 커서 대신 마크 주변의 픽셀 값이 표시됩니다. 마크는 원하는
만큼 만들 수 있으며, 각각 번호가 매겨진 「X」로 표시됩니다. 마크 드롭다운 컨트롤을
변경하기만 하면 다른 마크를 선택하여 그 주변의 값을 볼 수 있습니다. 현재 선택된
마크는 다른 것과 다른 색상으로 표시됩니다.

마크는 새 이미지가 불러와져도 위치를 유지하며 새 이미지의 값을 표시합니다. 이런
방식으로 이미지가 자주 업데이트되는 경우에도 어떤 지점 주변 영역을 모니터링할 수
있습니다.

「마크로 이동」 체크박스가 선택되어 있으면 마크 컨트롤에서 다른 마크를 선택할 때
채널 뷰어가 그 마크로 이동합니다. 이는 특히 이미지에 바짝 확대한 경우 여러 다른
이미지에서 같은 지점을 검사하는 데 유용할 수 있습니다.

.. note:: 마크 컨트롤을 다시 「None」으로 바꾸면 뷰어에서 커서를 움직일 때마다 픽셀
          테이블이 다시 업데이트됩니다.

「Caption」 상자는 다음 마크가 생성될 때 마크 레이블에 추가될 텍스트 주석을
설정하는 데 사용할 수 있습니다. 이는 예를 들어 이미지의 특징에 레이블을 다는 데
사용할 수 있습니다.

**마크 삭제**

마크를 삭제하려면 마크 컨트롤에서 선택한 다음 「삭제」라고 표시된 버튼을 누르십시오.
모든 마크를 삭제하려면 「모두 삭제」라고 표시된 버튼을 누르십시오.

**마크 이동**

「이동」 라디오 버튼이 선택되고 마크가 선택되어 있을 때 이미지의 아무 곳이나
클릭하거나 끌면 마크가 그 위치로 이동하고 픽셀 테이블이 업데이트됩니다. 현재 선택된
마크가 없으면 새 마크가 생성되어 이동됩니다.

**마크 그리기**

「그리기」 라디오 버튼이 선택되어 있을 때 클릭하고 끌면 새 마크가 생성됩니다. 끄는
길이가 길수록 「X」의 반지름이 커집니다.

**마크 편집**

마크를 선택한 후 「편집」 라디오 버튼이 선택되어 있으면 마크의 제어점을 끌어 X 팔의
반지름을 늘리거나, 경계 상자를 끌어 마크를 이동할 수 있습니다. 편집 제어점이
표시되지 않으면 마크의 중심을 클릭하기만 하면 활성화됩니다.

**특수 키**

「이동」 모드에서는 다음 키가 활성화됩니다:
- 「n」은 커서 위치에 새 마크를 놓습니다
- 「m」은 현재 마크(있는 경우)를 커서 위치로 이동합니다
- 「d」는 현재 마크(있는 경우)를 삭제합니다
- 「j」는 이전 마크(있는 경우)를 선택합니다
- 「k」는 다음 마크(있는 경우)를 선택합니다

**사용자 구성**

``~/.ginga/plugin_PixTable.cfg`` 를 사용하여 사용자 지정할 수 있으며, 여기서
``~`` 는 사용자의 HOME 디렉터리입니다:

.. code-block:: Python

  #
  # PixTable plugin preferences file
  #
  # Place this in file under ~/.ginga with the name "plugin_PixTable.cfg"

  # Default font
  font = 'fixed'

  # Default font size
  fontsize = 12

  # default size for mark point radius
  mark_radius = 10

  # style of point to draw
  mark_style = 'cross'

  # color of non-selected marks
  mark_color = 'purple'

  # color of selected mark
  select_color = 'cyan'

  # whether to update the pixel table when moving a mark around
  drag_update = True
