``LoaderConfig`` 플러그인을 사용하면 다양한 콘텐츠를 Ginga에 불러오는 데 사용할
수 있는 파일 오프너를 구성할 수 있습니다.

등록된 파일 오프너는 파일 MIME 유형과 연결되며, 하나의 MIME 유형에 여러 오프너가
있을 수 있습니다. MIME 유형/오프너 쌍에 연결된 우선순위가 각 유형에 어떤 오프너가
사용될지 결정합니다. 우선순위 값이 가장 낮은 것이 사용됩니다. 동일한 낮은
우선순위를 가진 오프너가 둘 이상 있으면 Ginga에서 파일을 열 때 어떤 오프너를
사용할지 사용자에게 묻습니다. 이 플러그인을 사용하여 오프너 기본 설정을 지정하고
사용자의 $HOME/.ginga 구성 영역에 저장할 수 있습니다.

**플러그인 유형: 전역**

``LoaderConfig`` 는 전역 플러그인입니다. 하나의 인스턴스만 열 수 있습니다.

**사용법**

플러그인을 시작하면 등록된 모든 MIME 유형과 해당 유형에 등록된 오프너가, 각 MIME
유형/오프너 쌍에 연결된 우선순위와 함께 표시됩니다.

한 줄 이상을 선택하고 「우선순위:」라고 표시된 상자에 우선순위를 입력하십시오.
「설정」(또는 ENTER)을 눌러 해당 항목의 우선순위를 설정합니다.

.. note:: 숫자가 낮을수록 우선순위가 높습니다. 음수도 괜찮으며, 로더의 기본
          우선순위는 보통 0입니다. 예를 들어, 한 MIME 유형에 두 개의 로더를
          사용할 수 있고 한 우선순위가 -1, 다른 하나가 0으로 설정된 경우,
          사용자에게 선택을 요청하지 않고 -1인 것이 사용됩니다.


「저장」을 클릭하면 우선순위가 $HOME/.ginga/loaders.json 에 저장되어, 이후 프로그램
재시작 시 다시 불러와 사용됩니다.
