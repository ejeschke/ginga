
``PluginConfig`` 플러그인을 사용하면 메뉴에 표시되는 플러그인을 구성할 수
있습니다.

**플러그인 유형: 전역**

``PluginConfig`` 는 전역 플러그인입니다. 하나의 인스턴스만 열 수 있습니다.

**사용법**

``PluginConfig`` 는 Ginga에서 사용할 플러그인을 구성하는 데 사용됩니다. 각
플러그인에 대해 구성할 수 있는 항목은 다음과 같습니다:

* 활성화 여부(따라서 메뉴에 표시되는지 여부)
* 플러그인의 범주(메뉴 계층을 구성하는 데 사용됨)
* 플러그인이 열릴 작업 공간
* 전역 플러그인인 경우 참조 뷰어가 시작될 때 자동으로 시작되는지 여부
* 플러그인 이름을 숨길지 여부(플러그인 활성화 메뉴에 표시하지 않음)

``PluginConfig`` 가 시작되면 플러그인 표가 표시됩니다. 플러그인의 위 속성을
편집하려면 「편집」을 클릭하십시오. 그러면 표를 편집하는 대화 상자가 나타납니다.

구성하려는 각 플러그인에 대해 메인 표의 항목을 클릭한 다음 대화 상자에서 설정을
조정하고, 대화 상자의 「설정」을 클릭하여 변경 사항을 표에 반영하십시오.
「설정」을 클릭하지 않으면 표에서 아무것도 변경되지 않습니다. 구성 편집을 마치면
대화 상자의 「닫기」를 클릭하여 편집 대화 상자를 닫으십시오.

.. note:: 원본과 크기가 호환되는 작업 공간을 선택하지 않는 한 플러그인의 작업
          공간을 변경하는 것은 권장되지 않습니다. 플러그인이 올바르게 표시되지
          않을 수 있기 때문입니다. 확실하지 않으면 작업 공간을 변경하지 마십시오.
          또한 「Systems」 범주의 플러그인을 비활성화하면 예상되는 일부 기능이
          작동하지 않게 될 수 있습니다.


.. important:: 변경 사항이 Ginga 재시작 후에도 유지되도록 하려면 「저장」을
               클릭하여 설정을 저장하십시오(`$HOME/.ginga/plugins.json` 에).
               메뉴의 변경 사항(「category」 변경을 통한)을 보려면 Ginga를 다시
               시작하십시오. **플러그인 구성을 기본값으로 재설정하려면 이 파일을
               수동으로 제거하십시오**.
