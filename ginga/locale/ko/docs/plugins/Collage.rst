
콜라주 방법으로 이미지 모자이크를 만드는 플러그인입니다.

**플러그인 유형: 로컬**

``Collage`` 은 로컬 플러그인으로, 채널에 연결되어 있습니다.
각 채널마다 하나의 인스턴스를 열 수 있습니다.

**사용법**

이 플러그인은 사용자가 제공한 이미지를 이용하여 채널 뷰어에 모자이크 콜라주를
자동으로 만드는 데 사용됩니다. 콜라주에서 이미지의 위치는 왜곡 보정 없이 해당
WCS에 의해 결정됩니다. 이는 빠른 미리 보기 도구이며, 이미지 왜곡 등을 고려하는
이미지 드리즐링의 대체가 아닙니다.

콜라주는 Ginga 캔버스의 플롯으로만 존재합니다. 새로운 단일 이미지가 실제로 만들
어지지는 않습니다(그것을 원하면 「Mosaic」 플러그인 참조). 단일 이미지에 대해
작동할 것으로 예상하는 일부 플러그인은 콜라주에서 올바르게 작동하지 않을 수
있습니다.

새 콜라주를 만들려면 「새 콜라주」 버튼을 클릭하고 파일을 표시 창으로 끌어다
놓으십시오(예: `FBrowser` 플러그인에서 파일을 끌어올 수 있습니다). 이미지에는
작동하는 WCS가 있어야 합니다. 처음 처리된 이미지가 불러와지고, 그 WCS가 다른
타일의 방향을 지정하는 데 사용됩니다. 추가 파일을 끌어다 놓기만 하면 기존
콜라주에 새 이미지를 추가할 수 있습니다.

**컨트롤**

「방법」 컨트롤은 콜라주의 이미지를 모자이크하는 방법을 선택하는 데 사용됩니다.
'simple' 과 'warp' 의 두 가지 값이 있습니다:

- 'simple' 은 WCS에 따라 이미지를 회전하고 뒤집으려고 시도합니다. 정확도를
  희생한 빠른 방법입니다. 이미지를 기울여야 하는 시야 가장자리 부근의 왜곡은
  처리하지 않습니다.
- 'warp' 는 WCS를 사용하여 참조 이미지의 WCS에 따라 이미지의 각 픽셀을 완전히
  이동시킵니다. 이로 인해 이미지에 빈 픽셀이 남을 수 있으며, 주변 픽셀에서
  샘플링하여 채워집니다. 이는 단순한 방법보다 느리며, 시간은 이미지 크기에 비례
  하여 선형으로 증가합니다.

「콜라주 HDU」 버튼을 선택하면 `Collage` 이 끌어다 놓은 파일에서 처음 발견된 하나만
이 아니라 모든 이미지 HDU를 플롯하려고 시도합니다.

「이미지에 레이블」을 선택하면 플러그인이 각 이미지의 이름을 플롯된 각 타일 위에
그립니다.

「배경 맞추기」를 선택하면 각 타일의 배경이 처음 플롯된 타일의 중앙값을 기준으로
조정됩니다(일종의 대략적인 평활화).

「스레드 수」 상자는 데이터를 불러오는 데 스레드 풀에서 몇 개의 스레드를 사용할지
할당합니다. 여러 스레드를 사용하면 보통 많은 파일의 불러오기가 빨라집니다.

**`Mosaic` 플러그인과의 차이점**

- 모든 모자이크 내용을 담기 위한 큰 배열을 할당하지 않음
- 출력 FOV를 지정하거나 신경 쓸 필요가 없음
- 결과를 더 빨리 표시할 수 있음(구성 이미지에 약간 좌우됨)
- 일부 플러그인은 콜라주에서 올바르게 작동하지 않거나 더 느려짐
- 콜라주를 데이터 파일로 저장할 수 없음(다만 「ScreenShot」을 사용할 수 있음)

``~/.ginga/plugin_Collage.cfg`` 를 사용하여 사용자 지정할 수 있으며, 여기서 ``~``
는 사용자의 HOME 디렉터리입니다:

.. code-block:: Python

  #
  # Collage plugin preferences file
  #
  # Place this in file under ~/.ginga with the name "plugin_Collage.cfg"

  # Set to True when you want to collage image HDUs in a file
  collage_hdus = False

  # annotate images with their names
  annotate_images = False

  # Try to match backgrounds
  match_bg = False

  # Number of threads to devote to opening images
  num_threads = 4
