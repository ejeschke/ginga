``LoaderConfig`` プラグインを使うと、さまざまなコンテンツを Ginga に読み込むために
使えるファイルオープナーを構成できます。

登録されたファイルオープナーはファイルの MIME タイプに関連付けられ、単一の MIME
タイプに複数のオープナーが存在できます。MIME タイプとオープナーの組に関連付けた
優先度によって、各タイプにどのオープナーが使われるかが決まります。優先度の値が
最も低いものが使われます。同じ低い優先度のオープナーが複数ある場合、Ginga で
ファイルを開くときにどのオープナーを使うかがユーザーに尋ねられます。この
プラグインを使って、オープナーの設定を行い、ユーザーの $HOME/.ginga 構成領域に
保存できます。

**プラグインの種類: グローバル**

``LoaderConfig`` はグローバルプラグインです。開けるインスタンスは 1 つだけです。

**使い方**

プラグインを起動すると、登録されたすべての MIME タイプと、それらのタイプに登録
されたオープナーが、各 MIME タイプとオープナーの組に関連付けられた優先度と共に
表示されます。

1 つ以上の行を選択し、「優先度:」というラベルのボックスに優先度を入力します。
「設定」（または ENTER）を押して、それらの項目の優先度を設定します。

.. note:: 数値が小さいほど優先度が高くなります。負の数でも問題なく、ローダの
          既定の優先度は通常 0 です。例えば、ある MIME タイプに 2 つのローダが
          利用可能で、一方の優先度が -1、もう一方が 0 に設定されている場合、
          ユーザーに選択を求めることなく -1 のものが使われます。


「保存」をクリックすると、優先度が $HOME/.ginga/loaders.json に保存され、以降の
プログラム再起動時に再読み込みされて使われます。
