
コラージュ法で画像モザイクを作成するプラグインです。

**プラグインの種類: ローカル**

``Collage`` はローカルプラグインで、チャンネルに関連付けられています。
チャンネルごとに 1 つのインスタンスを開けます。

**使い方**

このプラグインは、ユーザーが提供する画像を用いて、チャンネルビューアに自動的に
モザイクコラージュを作成するために使います。コラージュ上の画像の位置は、歪み補正
なしの WCS によって決まります。これはクイックルックのツールであり、画像の歪みなど
を考慮する画像ドリズリングの代替ではありません。

コラージュは Ginga キャンバス上のプロットとしてのみ存在します。新しい単一画像が
実際に構築されるわけではありません（それが必要な場合は「Mosaic」プラグインを参照）。
単一画像に対して動作することを前提とする一部のプラグインは、コラージュでは正しく
動作しないことがあります。

新しいコラージュを作成するには、「新規コラージュ」ボタンをクリックし、表示
ウィンドウにファイルをドラッグします（例: `FBrowser` プラグインからファイルを
ドラッグできます）。画像は有効な WCS を持っている必要があります。最初に処理された
画像が読み込まれ、その WCS が他のタイルの向きを決めるために使われます。追加の
ファイルをドラッグするだけで、既存のコラージュに新しい画像を追加できます。

**コントロール**

「方法」コントロールは、コラージュ内の画像をモザイクする方法を選ぶために使います。
'simple' と 'warp' の 2 つの値があります:

- 'simple' は WCS に従って画像を回転・反転しようとします。正確さを犠牲にした高速な
  方法です。画像を歪ませるべき視野の端付近の歪みは扱いません。
- 'warp' は WCS を使って、参照画像の WCS に従って画像の各ピクセルを完全に移動させ
  ます。これにより画像に空のピクセルが残ることがあり、周囲のピクセルからサンプ
  リングして埋められます。これはシンプルな方法より遅く、時間は画像のサイズに比例
  して増加します。

「コラージュ HDU」ボタンをチェックすると、`Collage` は、ドラッグしたファイル内の
最初に見つかった 1 つだけでなく、すべての画像 HDU をプロットしようとします。

「画像にラベル」をチェックすると、プラグインは各画像の名前を、プロットされた各
タイルの上に描きます。

「背景を合わせる」をチェックすると、各タイルの背景が、最初にプロットされたタイルの
中央値を基準に調整されます（一種の粗い平滑化）。

「スレッド数」ボックスは、データの読み込みにスレッドプールからいくつのスレッドを
使うかを割り当てます。複数のスレッドを使うと、通常は多数のファイルの読み込みが
速くなります。

**`Mosaic` プラグインとの違い**

- モザイクの内容全体を保持する大きな配列を確保しない
- 出力 FOV を指定したり気にしたりする必要がない
- 結果の表示が速いことがある（構成画像に少し依存する）
- 一部のプラグインはコラージュで正しく動作しないか、遅くなる
- コラージュをデータファイルとして保存できない（ただし「ScreenShot」を使える）

``~/.ginga/plugin_Collage.cfg`` を使ってカスタマイズできます。``~`` はあなたの
HOME ディレクトリです:

.. code-block:: Python

  #
  # Collage plugin preferences file
  #
  # Place this in file under ~/.ginga with the name "plugin_Collage.cfg"

  # Set to True when you want to collage image HDUs in a file
  collage_hdus = False

  # annotate images with their names
  annotate_images = False

  # Try to match backgrounds
  match_bg = False

  # Number of threads to devote to opening images
  num_threads = 4
