``LoaderConfig`` 插件允许您配置可用于将各种内容加载到 Ginga 的文件打开器。

已注册的文件打开器与文件 MIME 类型相关联，单个 MIME 类型可以有多个打开器。与
MIME 类型/打开器配对相关联的优先级决定每种类型将使用哪个打开器——优先级值最低的
将决定使用哪个打开器。如果有多个打开器具有相同的低优先级，则在 Ginga 中打开文件
时会提示用户使用哪个打开器。此插件可用于设置打开器偏好，并将其保存到用户的
$HOME/.ginga 配置区域。

**插件类型：全局**

``LoaderConfig`` 是全局插件。只能打开一个实例。

**用法**

启动插件后，显示区将展示所有已注册的 MIME 类型以及为这些类型注册的打开器，并为
每个 MIME 类型/打开器配对显示相关联的优先级。

选择一行或多行，并在标有「优先级:」的框中为它们输入优先级；按「设置」（或 ENTER）
以设置这些项的优先级。

.. note:: 数字越小，优先级越高。负数也可以，加载器的默认优先级通常为 0。因此，
          举例来说，如果某个 MIME 类型有两个可用的加载器，一个优先级设为 -1，
          另一个设为 0，则会使用 -1 的那个，而不会请用户选择。


单击「保存」将优先级保存到 $HOME/.ginga/loaders.json，以便在后续重启程序时
重新加载并使用它们。
