通过构建合成图像来创建图像拼接的插件。

**插件类型：本地**

``Mosaic`` 是本地插件，这意味着它与某个通道相关联。
可以为每个通道打开一个实例。

**用法**

.. warning:: 这可能非常占用内存。

此插件用于使用用户提供的图像（例如，使用 ``FBrowser``），在通道中自动构建拼接
图像。
图像在拼接图中的位置由其 WCS 确定，不进行畸变校正。这旨在作为快速查看工具，而
非考虑图像畸变等的图像 drizzle 的替代品。拼接图仅存在于内存中，但您可以使用
``SaveImage`` 将其保存为 FITS 文件。

当拼接图从内存中移出时，它在 Ginga 中将不再可访问。为避免这种情况，您必须配置
会话，使 Ginga 数据缓存足够大（参见手册中的「Customizing Ginga」）。

要创建新的拼接图，请设置 FOV 并将文件拖到显示窗口上。图像必须具有可用的 WCS。
第一幅图像的 WCS 将用于确定其他图块的方向。

**与 `Collage` 插件的区别**

- 分配单个大数组来保存所有拼接内容
- 构建较慢，但操作大型结果图像可能更快
- 可以将拼接图保存为新的数据文件
- 用填充值（可以是 `NaN`）填充图块之间的值
