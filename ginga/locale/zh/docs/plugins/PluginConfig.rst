
``PluginConfig`` 插件允许您配置在菜单中可见的插件。

**插件类型：全局**

``PluginConfig`` 是全局插件。只能打开一个实例。

**用法**

``PluginConfig`` 用于配置将在 Ginga 中使用的插件。每个插件可配置的项目包括：

* 是否启用（因此是否显示在菜单中）
* 插件的类别（用于构建菜单层次结构）
* 插件将在其中打开的工作区
* 如果是全局插件，参考查看器启动时是否自动启动
* 是否应隐藏插件名称（不显示在插件激活菜单中）

``PluginConfig`` 启动时，会显示一个插件表。要编辑插件的上述属性，请单击
「编辑」，这将弹出一个用于编辑该表的对话框。

对于要配置的每个插件，请单击主表中的一个条目，然后在对话框中调整设置，再单击
对话框中的「设置」以将更改反映回表中。如果不单击「设置」，表中不会有任何更改。
编辑完配置后，单击对话框中的「关闭」以关闭编辑对话框。

.. note:: 除非您选择与原始工作区大小兼容的工作区，否则不建议更改插件的工作区，
          因为该插件可能无法正确显示。如有疑问，请保持工作区不变。此外，禁用
          「Systems」类别中的插件可能会导致某些预期功能停止工作。


.. important:: 要使更改在 Ginga 重启后仍然保留，请单击「保存」以保存设置（保存到
               `$HOME/.ginga/plugins.json`）。重启 Ginga 以查看对菜单的更改
               （通过「category」更改）。**如果要将插件配置重置为默认值，请手动
               删除此文件**。
